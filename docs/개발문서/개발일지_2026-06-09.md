# 개발일지 2026-06-09: 실적관리 대시보드 "실적 표" 참여자 3개년도 동적 조회

> **작성일:** 2026-06-09  
> **브랜치:** develop  
> **대상:** 실적관리 대시보드(지점별 실적) - 실적 표 참여자 연도별 집계

---

## 1. 배경 및 목적

실적관리 대시보드(`DashBoardBranchScoreAndSituation.jsp`)의 **실적 표**에는 참여자 수가 3개년도 컬럼(`참여자(2023)/(2024)/(2025)`)으로 표시된다. 다음 두 가지 문제가 있었다.

1. **SQL 집계가 `GETDATE()` 고정** — `selectBranchTable`/`selectPeopleTable`의 `참여자_년도별_집계` CTE가 오늘 날짜 기준으로만 3개년도를 산출하여, 사용자가 선택한 **실적 기간을 따르지 않았다.**
2. **헤더 연도 하드코딩** — 표 헤더가 `참여자(2023)/(2024)/(2025)`로 고정되어, 연도가 바뀐 현재(2026년) 데이터(2024/2025/2026)와 **불일치**하고 실적 기간을 변경해도 헤더가 변하지 않았다.

**요구사항:** 처음 접속 시 1번째=재작년, 2번째=작년, 3번째=당해 기준으로 조회하고, 실적 기간 선택에 따라 헤더·데이터 3개년도가 함께 이동해야 한다.

---

## 2. 설계 결정

**기준 연도 = 실적 기간 종료일(`dashBoardEndDate`) 연도 = 당해년도**

- 기본 실적 기간이 `(연도-1)-11-01 ~ 연도-10-31` 이므로 `YEAR(종료일)` = 선택 연도 = 당해. "3번째 당해년도" 요구와 정확히 일치.
- 재작년 = 기준-2, 작년 = 기준-1, 당해 = 기준.
- 헤더(JS/JSP)와 SQL이 **동일한 종료일 값**을 사용하므로 항상 일치한다.

---

## 3. 변경 내용 (코드 리뷰)

### 3-1. SQL — `Dashboard-mapping.xml`

`selectBranchTable`(지점별)·`selectPeopleTable`(개인별)의 `참여자_년도별_집계` CTE 연도 버킷을 `GETDATE()` 기반 → 기준 연도 파라미터(`#{dashBoardBaseYear}`) 기반으로 교체.

```sql
-- 변경 전: SUM(CASE WHEN 등록년도 = YEAR(GETDATE()) THEN 1 ELSE 0 END) AS 참여자_2025
-- 변경 후
SUM(CASE WHEN 등록년도 = #{dashBoardBaseYear} - 2 THEN 1 ELSE 0 END) AS 참여자_2023,
SUM(CASE WHEN 등록년도 = #{dashBoardBaseYear} - 1 THEN 1 ELSE 0 END) AS 참여자_2024,
SUM(CASE WHEN 등록년도 = #{dashBoardBaseYear}     THEN 1 ELSE 0 END) AS 참여자_2025,
SUM(CASE WHEN 등록년도 IN (#{dashBoardBaseYear} - 2, #{dashBoardBaseYear} - 1, #{dashBoardBaseYear}) THEN 1 ELSE 0 END) AS 참여자_합계
```

- `등록년도`(=`YEAR(...)`, int)와 `#{dashBoardBaseYear}`(int) 간 산술이라 T-SQL에서 정상 동작.
- 컬럼 별칭 `참여자_2023/2024/2025`는 의미상 "기준-2/-1/기준"으로 바뀌었으나, SELECT/GROUP BY/DTO 매핑 변경 churn을 줄이기 위해 **이름은 유지**(실제 표기는 JSP 헤더가 결정).

### 3-2. DTO — `DashboardDTO.java`

기준 연도 전달용 필드 추가. (Lombok `@Data` → getter/setter 자동 생성)

```java
private int dashBoardBaseYear; // 실적 표 참여자 3개년도 기준 연도(= 실적 기간 종료일 연도, 당해년도)
```

### 3-3. 컨트롤러 — `DashboardAjaxController.java` (`scoreBranchPerformanceTableAjax`)

서비스 호출 전 `dashBoardEndDate`에서 기준 연도를 파싱하여 DTO에 설정. null/공백/형식오류 시 현재 연도로 폴백(방어적 검증).

```java
String endDate = dashboardDTO.getDashBoardEndDate();
int baseYear;
if (endDate != null && endDate.length() >= 4) {
    try { baseYear = Integer.parseInt(endDate.substring(0, 4)); }
    catch (NumberFormatException e) { baseYear = Calendar.getInstance().get(Calendar.YEAR); }
} else { baseYear = Calendar.getInstance().get(Calendar.YEAR); }
dashboardDTO.setDashBoardBaseYear(baseYear);
```

- 실적 표 AJAX 경로만 `참여자_년도별_집계` CTE를 사용하므로 그래프/지점점수 AJAX는 변경 불필요.

### 3-4. 프론트엔드 — `DashBoardBranchScoreAndSituation.jsp`

- `fn` taglib 추가 후 헤더를 종료일 기준 JSTL로 동적 렌더(하드코딩 제거):
  ```jsp
  <c:set var="baseYear" value="${fn:substring(dashBoardEndDate, 0, 4)}" />
  <th id="dashboardByYearCount1-th">참여자(${baseYear - 2})</th> ... 참여자(${baseYear})
  ```
- 실적 기간 변경 시 헤더 동기화하는 `updateYearHeaders()` 추가, `#dashBoardStartDate/#dashBoardEndDate` `change` 이벤트에 바인딩.
- 표 fetch 함수(`fetchPerformanceTableContract`)는 헤더를 건드리지 않아 정렬 아이콘(`<span id="sort">`)과 충돌하지 않음.

## 4. 이번 업데이트 요약

- 실적 표의 참여자 3개년도가 **선택한 실적 기간(종료일 연도)에 따라 동적으로 조회·표기**되도록 변경.
- 첫 접속 시 재작년/작년/당해(예: 2024/2025/2026)로 정상 표시되며, 종료일 변경 시 헤더·데이터가 함께 이동.
- 연도 롤오버 시 헤더-데이터 불일치 버그 해소.
