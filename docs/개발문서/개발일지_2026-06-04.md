## 1. 배경 및 목적

추천 채용정보 모달에서 추천 후보군 테이블을 확인할 때 경력과 근무지역 정보를 바로 확인할 수 없어 상담사가 별도로 공고 상세를 열어봐야 했다. 기존 DB(`J_참여자관리_JOB_POSTING`)에 `career`, `region_nm` 컬럼이 이미 존재하고 `selectRecommendList` 쿼리가 해당 테이블을 LEFT JOIN하고 있어, SELECT 절에 컬럼만 추가하면 프론트엔드까지 자동으로 전달되는 구조였다. 추가된 컬럼을 수용하기 위해 모달 크기도 함께 확대했다.

## 2. Backend 구현

### 2.1 DTO 필드 추가

| 파일 | 변경내용 |
|------|----------|
| `ParticipantJobRecommendDTO.java` | `career` (경력), `regionNm` (근무지역) 필드 2개 추가 |

- `@Data` (Lombok)이 getter/setter 자동 생성 → DAO/Service/Controller 변경 불필요
- Jackson이 DTO→JSON 자동 직렬화 → API 응답에 자동 포함

### 2.2 MyBatis SQL 수정

| 파일 | 변경내용 |
|------|----------|
| `ParticipantJobRecommend-mapping.xml` | `selectRecommendList` SELECT 절에 `P.career AS career`, `P.region_nm AS regionNm` 추가 |

- 기존 LEFT JOIN (`J_참여자관리_JOB_POSTING P ON R.추천채용정보_구인인증번호 = P.wanted_auth_no`) 그대로 활용
- JOIN 절 변경 불필요
