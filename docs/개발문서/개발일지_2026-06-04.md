## 파트 A: 추천 채용정보 모달 크기 확대 및 경력/근무지역 컬럼 추가

### 1. 배경 및 목적

추천 채용정보 모달에서 추천 후보군 테이블을 확인할 때 경력과 근무지역 정보를 바로 확인할 수 없어 상담사가 별도로 공고 상세를 열어봐야 했다. 기존 DB(`J_참여자관리_JOB_POSTING`)에 `career`, `region_nm` 컬럼이 이미 존재하고 `selectRecommendList` 쿼리가 해당 테이블을 LEFT JOIN하고 있어, SELECT 절에 컬럼만 추가하면 프론트엔드까지 자동으로 전달되는 구조였다. 추가된 컬럼을 수용하기 위해 모달 크기도 함께 확대했다.

### 2. 변경 내용

| 파일 | 변경내용 |
|------|----------|
| `ParticipantJobRecommendDTO.java` | `career`, `regionNm` 필드 추가 |
| `ParticipantJobRecommend-mapping.xml` | `selectRecommendList`에 `P.career`, `P.region_nm` 추가 |
| `recommendModal.jsp` | 테이블 헤더 경력/근무지역 추가 |
| `recommend-modal_0.0.5.css` (신규) | 모달 width `min(1400px, 96vw)` |
| `recommend-modal_0.0.8.js` (신규) | 테이블 셀 career/regionNm 추가 |

## 파트 B: AI 추천 채용정보 정확도 전면 개선 (Phase 1~5)

### 1. 배경 및 목적

기존 AI 추천 시스템은 참여자 데이터를 충분히 활용하지 않고(자격증·직업훈련·추천키워드 미사용), 프롬프트가 느슨하며, 검색 쿼리가 관련성 기반이 아닌 OR 조건의 광범위 검색이라 추천 정확도에 한계가 있었다. 전체 5단계에 걸쳐 프롬프트, 데이터, 검색 로직, 모델 구조, 테스트를 개선했다.

### 2. Phase별 구현 내용

#### Phase 1 — 프롬프트/설정 개선 (커밋 `6d332a9`)

| 항목 | 기존 | 개선 |
|------|------|------|
| Stage 1 프롬프트 | 역할 없음, 기본 정보만 전달 | 역할 부여("공공고용서비스 전문 상담사"), 추천사 추가, 키워드 가이드(3~7개, 일반어 제외), 거주지 기반 지역 검색 |
| Stage 2 프롬프트 | 평가 기준 5개 동일 가중치 | 5항목 가중치(직무30/자격25/학력20/경력15/급여10), 추천사 전달, 할루시네이션 유도 문구 삭제 |
| 지역 검색 | 알선상세정보에 희망지역 있을 때만 | 1순위: 희망지역 → 2순위: 거주지 기반 인접 지역 자동 추정 |
| temperature | 미설정 (매번 다른 결과) | 0.1 (일관된 결과) |
| Stage 2 실행 | 알선상세정보 있을 때만 | 항상 실행 (모든 참여자 점수 산출) |
| 모델 | gemini-2.5-flash-lite | gemini-2.5-flash |

**주요 파일:** `GeminiApiService.java`, `ParticipantJobRecommendServiceImpl.java`

#### Phase 2 — 데이터 보강 (커밋 `8dbacd0`)

| 추가 데이터 | 출처 테이블 | 활용 |
|-------------|-------------|------|
| 보유 자격증 | `J_참여자관리_자격증` | Stage 1/2 프롬프트에 반영, 자격증→직무 키워드 변환 가이드 |
| 직업훈련 이력 | `J_참여자관리_직업훈련` | Stage 1/2 프롬프트에 반영 |
| 특정계층 | `J_참여자관리.특정계층` | 프롬프트에 반영 |
| 취업역량 | `J_참여자관리.취업역량` | 프롬프트에 반영 |
| 추천키워드 | `J_참여자관리_희망직무.추천키워드` | 검색 조건 최우선 반영 지시 |

**폴백 검색조건:** AI 검색조건 생성 실패(파싱 오류/키워드 비어있음) 시 참여자 데이터에서 직접 키워드 추출하는 `buildFallbackSearchCondition()` 메서드 추가. 추천키워드 → 희망직무 → 카테고리 중분류 → 자격증명 순서로 키워드 조립, 주소에서 광역/기초 단위 직접 파싱.

**주요 파일:** `ParticipantJobRecommend-mapping.xml`, `ParticipantJobRecommendDAO.java`, `RecommendParticipantDTO.java`, `RecommendCategoryDTO.java`

#### Phase 3 — 검색 쿼리 최적화 (커밋 `c7cc864`)

| 항목 | 기존 | 개선 |
|------|------|------|
| 검색 컬럼 | `recrut_title`, `ind_tp_nm`, `keyword_list` (3개) | + `job_cont`, `rel_jobs_nm`, `pref_cond` (6개) |
| 지역 매칭 | `basic_addr IN ('서울')` 정확 매칭 | `region_nm LIKE '%서울%'` 부분 매칭 |
| 정렬 | `sync_dtm_detail DESC` (최신순) | 관련성 점수(직종코드3 + 키워드2 + 지역1) DESC → 최신순 |
| 0건 처리 | 즉시 "찾지 못했습니다" 반환 | 완화 재검색 자동 실행 (지역 제거, 키워드 2개만) |

**주요 파일:** `ParticipantJobRecommend-mapping.xml`, `ParticipantJobRecommendDAO.java`, `ParticipantJobRecommendServiceImpl.java`

#### Phase 4 — 모델/구조 개선 (커밋 `1063b8d`)

**주요 파일:** `GeminiApiService.java`, `application.properties`, `.env`
