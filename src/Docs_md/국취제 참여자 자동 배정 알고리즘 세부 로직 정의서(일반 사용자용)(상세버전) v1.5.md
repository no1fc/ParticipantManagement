# 국취제 참여자 자동 배정 알고리즘 세부 로직 정의서(일반 사용자용)(상세버전) v1.5

## 1. 문서 목적
이 문서는 국취제 참여자 자동 배정 화면의 사용 방법과 자동 배정 계산 방식(가중치/제한/균형 로직)을 일반 사용자 관점에서 상세하게 설명한다.

## 2. 화면 구성 요약
- 참여자 입력 영역: CSV 업로드, 직접 입력
- 산정식 설정 카드: 가중치/제한값 입력
- 상담사 통계 테이블(assign-count-table)
- 배정 결과 테이블(csv-data)
- 버튼: 랜덤 배정, 데이터 검증, 저장, 초기화, 엑셀 다운로드

## 3. 기본 사용 흐름(권장)
1) CSV 업로드 또는 직접 입력으로 참여자 목록 준비
2) 산정식 설정(가중치/제한)을 확인 또는 조정
3) 랜덤 배정 실행
4) 데이터 검증(오류 확인)
5) 저장

## 4. CSV 업로드 상세
### 4.1 CSV 컬럼 순서
1) 참여자 성명
2) 참여유형
3) 성별
4) 생년월일
5) 모집경로
6) 경력유무(숫자)
7) 학력
8) 특정계층여부
9) 진행단계
10) 출장상담사

### 4.2 입력 규칙
- 참여유형: 1, 2, 1유형, 2유형
- 성별: 남/여
- 생년월일: YYYY-MM-DD
- 경력유무: 숫자만 입력
- 특정계층여부: O/X/Y/N/해당/해당 없음
- 출장상담사: 선택 입력(입력 시 고정 배정)

### 4.3 출장상담사 입력 시 동작
- 입력된 상담사ID로 해당 참여자가 **고정 배정**된다.
- 고정 배정은 랜덤 배정 전에 우선 적용된다.

## 5. 직접 입력(수동 입력)
- CSV 없이 1건씩 추가 입력 가능
- “목록에 추가” 버튼으로 하단 테이블에 반영
- 출장상담사 선택 시 고정 배정

## 6. 상담사 통계 테이블 이해
- 상담사별 배정 현황이 표시됨
- 주요 컬럼
  - 유형1/유형2: 참여유형별 누적 배정 건수
  - 일일/주간/2주/월간 배정 건수
  - 청년/중장년/특정계층 배정 건수
- 이 중 **유형1/유형2**는 일일업무보고 히스토리로 저장된다.

## 7. 산정식 설정(가중치) 상세
### 7.1 가중치의 의미
가중치는 “어떤 기준을 더 중요하게 볼지” 결정한다.

- C_load: 현재 담당 건수가 많으면 점수가 커져 배정 우선순위가 낮아짐
- C_fair: 유형/성별/연령대/특정계층 분포가 균형에서 멀수록 점수 증가
- C_streak: 연속 배정이 많을수록 점수 증가
- C_pace: 연간 배정 편차가 클수록 점수 증가

### 7.2 제한값의 의미
- Gap 임계값: 근속 그룹 내 편차가 커지면 적게 배정된 상담사를 우선 고려
- 일일 제한: 하루 최대 배정 건수
- G1/G2/G3 제한: 근속기간 그룹별 7/14/30일 배정 제한

### 7.3 근속 그룹
- G1: 입사 3개월 이하
- G2: 입사 4~11개월
- G3: 입사 12개월 이상

## 8. 랜덤 배정 계산식(상세)
### 8.1 기본 계산식
상담사별 점수를 계산해 **점수가 가장 낮은 상담사**에게 배정한다.

```
총점 = (C_load * 가중치로드)
     + (C_fair * 가중치공정)
     + (C_streak * 가중치연속)
     + (C_pace * 가중치페이스)
```

### 8.2 각 항목의 계산 방향
- C_load: 현재 담당 건수와 같은 근속 그룹 평균의 차이가 클수록 점수 증가
- C_fair: 참여유형/성별/연령대/특정계층 분포가 균형에서 벗어날수록 점수 증가
- C_streak: 연속 배정이 길수록 점수 증가
- C_pace: 연간 배정 건수와 같은 그룹 평균의 차이가 클수록 점수 증가

즉, **점수가 낮은 상담사**가 우선 배정되며, 가중치가 높을수록 해당 기준의 영향이 커진다.

### 8.3 점수 동점 처리
점수가 같으면 다음 순서로 우선순위를 정한다.
1) 현재 진행 건수가 더 적은 상담사
2) 최근 7일 배정 건수가 더 적은 상담사
3) 그래도 같으면 무작위

## 9. 균형/강제 배정 로직
### 9.1 강제 배정(편차 보정)
- 같은 근속 그룹 내에서 상담사별 진행 건수 편차가 **Gap 임계값 이상**이면,
  해당 그룹의 **가장 적게 배정된 상담사**에게 우선 배정한다.

### 9.2 제한 리셋
- 모든 상담사가 7/14일 제한에 도달한 경우,
  한시적으로 1/7/14/30일 카운터를 초기화하고 다시 배정한다.

## 10. 미배정 처리
- 조건을 만족하는 상담사가 없으면 미배정으로 남을 수 있다.
- 저장 시 미배정 인원이 있으면 경고 후 진행 여부를 확인한다.

## 11. 데이터 검증
- “데이터 검증” 버튼으로 입력 오류를 사전에 확인 가능
- 오류가 있으면 저장/배정을 중단하고 오류 목록을 표시

## 12. 저장 시 기록되는 내용
저장 시 아래 내용이 함께 기록된다.
- 배정 결과
- 일일업무보고(유형1/유형2)
- 산정식 설정값
- CSV 전체 입력 내용

## 13. 자주 묻는 질문
Q. 배정이 안 된 참여자가 있으면?
A. 저장 전 경고가 표시되며, 확인 후 진행 여부를 선택할 수 있다.

Q. 가중치를 올리면 어떻게 되나?
A. 해당 기준의 영향이 커져 그 기준을 더 많이 고려한다.

Q. 산정식 설정은 언제 적용되나?
A. 다음 “랜덤 배정” 실행 시 적용된다.
