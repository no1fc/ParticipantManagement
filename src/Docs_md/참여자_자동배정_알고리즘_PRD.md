# 참여자 자동배정 알고리즘 PRD

## 1. 문서 목적
- 참여자 자동배정 알고리즘의 제품 요구사항을 정의한다.
- 운영(고용센터/관리자)과 상담사 관점에서 기대되는 동작과 품질 기준을 명확히 한다.

## 2. 배경 및 문제 정의
- 배정 편향으로 인한 상담사별 진행 인원 격차가 발생한다.
- 참여자 속성이 특정 상담사에 집중되어 형평성이 저하된다.
- 연간 배정 페이스가 불균형해 업무 과부하가 발생한다.

## 3. 목표
- 경력 그룹 내 진행 인원 편차를 ±5명 이내로 유지한다.
- 참여자 속성 분산을 유도하여 형평성을 높인다.
- 연간 배정 페이스를 그룹 평균에 수렴하도록 관리한다.

## 4. 범위
### In Scope
- 상담사 가용성 및 배정 한도를 고려한 후보군 필터링
- 경력 그룹 불균형 감지 및 강제 배정 모드
- 비용 함수 기반 자동 최적 배정
- 배치(다중 참여자) 순차 배정 및 동적 분산
- 동점자 처리 및 상태 갱신

### Out of Scope
- 상담 내용 품질 평가 및 성과 분석
- 상담사/참여자 매칭의 정성적 추천
- 외부 시스템 통합 자동화(별도 프로젝트)

## 5. 사용자 및 이해관계자
- 관리자(고용센터): 배정 정책 설정, 운영 모니터링
- 상담사: 배정된 참여자 처리, 업무량 균형 확인
- 시스템 운영팀: 배정 로직 유지보수 및 장애 대응

## 6. 사용자 스토리
- 관리자로서, 상담사 근무 상태를 지정하고 배정 정책에 따라 자동 배정을 실행하고 싶다.
- 관리자로서, 배정 결과가 공정하고 설명 가능하길 원한다.
- 상담사로서, 내 진행 인원이 과도하게 증가하지 않길 원한다.
- 상담사로서, 특정 유형의 참여자가 편중되지 않길 원한다.

## 7. 기능 요구사항
### 7.1 후보군 생성 (Hard Constraints)
- 근무 가능 상태인 상담사만 후보로 포함한다.
- 일일 배정수(`assigned_1d`)는 3명 미만이어야 한다.
- 경력 그룹별 기간 한도(7/14/30일)를 초과하지 않아야 한다.

### 7.2 불균형 트리거
- 경력 그룹 내 `active_case_count`의 격차가 5명 이상이면 강제 배정 모드를 활성화한다.
- 강제 배정 모드에서는 그룹 내 진행 인원이 최소인 상담사만 후보로 제한한다.

### 7.3 비용 함수 기반 배정
- 강제 배정이 아닌 경우, 다음 비용 함수를 계산한다.
  - `TotalCost = 0.45*C_load + 0.35*C_fair + 0.10*C_streak + 0.10*C_pace`
- 최소 비용 상담사를 배정 대상으로 선택한다.

### 7.4 동점자 처리
- 비용이 동일하면 `active_case_count` 적은 순 → `assigned_7d` 적은 순 → 랜덤 순으로 결정한다.

### 7.5 상태 갱신
- 배정 확정 시 `active_case_count`, 기간별 배정수, 속성 카운트, `streak_count`를 즉시 갱신한다.

### 7.6 배치 처리
- 참여자 리스트는 난이도 높은 순으로 정렬한다.
- 정렬된 순서대로 단건 배정을 반복한다.
- 배정 중간마다 상담사 상태를 갱신해 동적 분산 효과를 반영한다.

## 8. 데이터 요구사항
- 상담사 상태: `active_case_count`, `assigned_1d/7d/14d/30d`, `assigned_ytd`, `streak_count`
- 상담사 메타: 경력 그룹(G1/G2/G3), 근무 가능 상태
- 참여자 속성: 참여유형, 성별, 연령대, 경력유무, 학력, 특정계층 여부, 진행단계, 출장 여부

## 9. 비기능 요구사항
- 결정 과정이 설명 가능해야 한다(후보군, 비용 함수, 동점자 기준 로그).
- 동일 입력에 대해 결정 재현 가능해야 한다(랜덤은 시드 관리).
- 배치 처리 시 처리 순서와 결과가 추적 가능해야 한다.

## 10. 성공 지표
- 경력 그룹 내 진행 인원 편차 평균이 ±5명 이내로 유지되는 비율
- 특정 속성 편중 비율 감소
- 연간 배정 페이스의 표준편차 감소

## 11. 리스크 및 대응
- 데이터 지연으로 상태 정보가 늦게 반영될 위험 → 상태 갱신 트랜잭션 보장
- 희귀 속성 참여자가 특정 상담사로 집중될 위험 → `C_fair` 가중치 검토
- 배정 불가 케이스 발생 → 대기열 또는 수동 배정 프로세스 필요

## 12. 가정 및 의존성
- 상담사/참여자 속성 데이터가 실시간으로 정확하다.
- 배정 로직에 필요한 상태 변수가 저장 및 조회 가능하다.

## 13. 오픈 이슈
- 강제 배정 모드에서 동점자 기준에 `assigned_7d` 외 추가 지표 필요 여부
- 희귀 속성 난이도 점수 산정 방식 확정 필요
- 대기열 정책(재시도 주기/우선순위) 정의 필요
