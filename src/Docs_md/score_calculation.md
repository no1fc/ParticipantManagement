## 1. 배정 흐름 요약

참여자 1명당 다음 순서로 상담사가 배정됩니다:

```
참여자 입력
    ↓
[1단계] 후보 상담사 목록 구성
    ↓
[2단계] 후보별 비용(cost) 계산
    ↓
[3단계] 비용 → 점수(score) 변환
    ↓
[4단계] 점수 상위 4명 선정
    ↓
[5단계] 한도 만족 필터링 (랜덤 배정 모드만)
    ↓
[6단계] 배정인원 최소인 상담사 우선
    ↓
[7단계] 최종 후보 중 랜덤 선택
    ↓
배정 완료
```

## 2. 배정 모드별 차이

시스템은 두 가지 배정 모드를 제공합니다:

### 2.1 랜덤 배정 (일반 모드)

**특징:**
- ✅ 한도 제한 적용 (일/주/2주/월)
- ✅ 불균형 임계값 확인
- ✅ 공정한 배정 분산

**후보 선정 조건:**
1. 정원 초과하지 않음
2. 상담사 가용 상태 활성
3. 일일 한도 미초과
4. 주간 한도 미초과
5. 2주간 한도 미초과
6. 월간 한도 미초과

**사용 시나리오:** 일반적인 참여자 배정

---

### 2.2 점수만 반영 배정

**특징:**
- ❌ 한도 제한 무시
- ✅ 순수 점수 기반 배정
- ⚠️ 불균형 발생 가능

**후보 선정 조건:**
1. 정원 초과하지 않음
2. 상담사 가용 상태 활성

**사용 시나리오:**
- 긴급 배정이 필요한 경우
- 한도를 일시적으로 무시해야 하는 경우
- 순수 점수만으로 최적 매칭을 원하는 경우

## 3. 후보 상담사 구성

### 3.1 공통 조건

모든 배정 모드에서 다음 조건을 만족해야 합니다:

| 조건 | 설명 |
|------|------|
| **정원 확인** | `배정인원(total) < 최대정원(max)` |
| **가용 상태** | `isAvailable = true`, `available = true`, `onDuty = true` |

### 3.2 랜덤 배정 모드 추가 조건

| 한도 유형 | 확인 항목 |
|-----------|-----------|
| **일일 한도** | `일일배정수 < 일일한도` |
| **주간 한도** | `주간배정수 < 근속그룹별 주간한도` |
| **2주간 한도** | `2주간배정수 < 근속그룹별 2주간한도` |
| **월간 한도** | `월간배정수 < 근속그룹별 월간한도` |

**근속그룹별 기본 한도:**
- **G1 (신입)**: 주 3명 / 2주 6명 / 월 12명
- **G2 (중견)**: 주 5명 / 2주 10명 / 월 14명
- **G3 (숙련)**: 주 8명 / 2주 15명 / 월 15명

## 4. 점수 계산식 상세

점수 계산은 4가지 비용(Cost) 요소를 종합하여 계산됩니다.

### 4.1 용어 정의

| 용어 | 설명 |
|------|------|
| `active` | 상담사의 현재 활성 케이스 수 (백엔드 제공) |
| `ytd` | 상담사의 연간 누적 배정 수 |
| `total` | 상담사의 전체 배정 인원 (백엔드의 `assignmentTotal`) |
| `overallAvg.activeAvg` | 전체 후보의 평균 활성 케이스 수 |
| `overallAvg.ytdAvg` | 전체 후보의 평균 연간 배정 수 |
| `streak` | 연속 배정 횟수 |

### 4.2 비용 계산 항목

#### 📊 1) 업무량 비용 (C_load)

현재 진행 중인 케이스 수의 편차를 측정합니다.

```javascript
cLoad = (active - overallAvg.activeAvg)²
```

**의미:**
- 평균보다 많은 케이스를 담당 중 → 비용 증가 (배정 가능성 감소)
- 평균보다 적은 케이스를 담당 중 → 비용 감소 (배정 가능성 증가)

---

#### ⚖️ 2) 공정성 비용 (C_fair)

참여자 특성별로 상담사의 배정 비율을 확인하여 편향을 방지합니다.

**계산 방식:**
```javascript
cFair = 평균([유형비율, 성별비율, 연령비율, 특정계층비율*, 학력비율*, 경력비율*])
```
(*: 해당되는 경우만 포함)

**상세 계산:**
```
유형비율 = (상담사의 해당 유형 배정 수) / (상담사 총 배정 수)
성별비율 = (상담사의 해당 성별 배정 수) / (상담사 총 배정 수)
연령비율 = (상담사의 해당 연령 배정 수) / (상담사 총 배정 수)
특정계층비율 = (상담사의 특정계층 배정 수) / (상담사 총 배정 수)
학력비율 = (상담사의 해당 학력 배정 수) / (상담사 총 배정 수)
경력비율 = (상담사의 해당 경력 배정 수) / (상담사 총 배정 수)
```

**특성별 분류:**
- **참여유형**: Type1(청년) / Type2(중장년)
- **성별**: 남성 / 여성
- **연령**: 청년(15-34세) / 중장년(35세 이상)
- **특정계층**: 해당 / 미해당
- **학력**: 대졸 그룹(초대졸, 대졸, 대학원) / 고졸 그룹(고졸, 검정고시)
- **경력**: 경력 있음 / 경력 없음

**의미:**
- 특정 특성에 편향된 배정 → 비용 증가
- 균형잡힌 배정 분포 → 비용 감소

---

#### 🔄 3) 연속 배정 비용 (C_streak)

동일 상담사에게 연속으로 배정되는 것을 방지합니다.

```javascript
cStreak = (streak)²
```

**의미:**
- 연속 배정 횟수가 많을수록 비용 증가
- 다른 상담사에게 배정 기회 제공

---

#### ⏱️ 4) 진행 속도 비용 (C_pace)

연간 배정 속도의 편차를 측정합니다.

```javascript
cPace = (ytd - overallAvg.ytdAvg)²
```

**의미:**
- 평균보다 많이 배정받음 → 비용 증가
- 평균보다 적게 배정받음 → 비용 감소

---

### 4.3 가중치 합산

4가지 비용을 가중치로 합산하여 최종 비용을 계산합니다.

```javascript
cost = (w_load × cLoad) + (w_fair × cFair) + (w_streak × cStreak) + (w_pace × cPace)
```

**기본 가중치 (초기값):**
| 항목 | 가중치 | 설명 |
|------|--------|------|
| `w_load` | 0.45 | 업무량 비중 (45%) |
| `w_fair` | 0.35 | 공정성 비중 (35%) |
| `w_streak` | 0.10 | 연속 배정 방지 (10%) |
| `w_pace` | 0.10 | 진행 속도 (10%) |

**가중치 조정:**
화면에서 실시간으로 가중치를 조정할 수 있습니다.

---

### 4.4 인원 가중치 보정

백엔드에서 전달되는 상담사별 인원 가중치를 적용합니다.

```javascript
headcountWeight = clamp(0.5 ~ 2.0)
adjustedCost = cost / headcountWeight
```

**의미:**
- 인원 가중치 높음(2.0) → 더 많은 인원 배정 필요 → 비용 감소
- 인원 가중치 낮음(0.5) → 적은 인원 배정 필요 → 비용 증가

---

### 4.5 비용 → 점수 변환

비용이 낮을수록 점수가 높아지도록 역함수로 변환합니다.

```javascript
score = 1 / (1 + adjustedCost)
```

**점수 범위:** 0 < score ≤ 1

**의미:**
- 비용 0 → 점수 1.0 (최고)
- 비용 증가 → 점수 감소

## 5. 최종 배정 규칙

점수 계산 후 다음 순서로 최종 배정이 이루어집니다:

### 5.1 배정 단계

```
[1단계] 점수 계산 완료
    ↓
[2단계] 점수 상위 4명 선정
    ↓
[3단계] 한도 만족 필터링
    ├─ 랜덤 배정 모드: 한도 만족 상담사 우선
    └─ 점수만 반영 모드: 필터링 스킵
    ↓
[4단계] 배정인원 최소인 상담사 선정
    ↓
[5단계] 동점자 중 랜덤 선택
    ↓
배정 완료
```

### 5.2 상세 규칙

#### ✅ 1단계: 점수 상위 4명 선정

- 모든 후보 상담사의 점수를 계산
- 점수가 높은 순으로 **상위 4명** 선정
- 후보가 4명 미만인 경우 전체 선정

#### ✅ 2단계: 한도 만족 필터링 (랜덤 배정 모드만)

**랜덤 배정 모드:**
- 상위 4명 중 일/주/2주/월 한도를 **모두 만족**하는 상담사만 필터링
- 한도를 만족하는 상담사가 없으면 상위 4명 전체를 유지

**점수만 반영 배정 모드:**
- 이 단계를 **건너뜀** (한도 무시)

#### ✅ 3단계: 배정인원 최소인 상담사 선정

- 필터링된 후보 중 **전체 배정 인원**(total)이 가장 적은 상담사 선정
- `total` = 백엔드에서 받은 `assignmentTotal` + 현재 배정 프로세스에서 증가한 수
- 동일한 배정인원을 가진 상담사가 여러 명이면 모두 최종 후보로 유지

#### ✅ 4단계: 랜덤 선택

- 최종 후보 중 **무작위로 1명** 선택
- 동점자가 여러 명인 경우 공정하게 분산

## 6. 불균형 임계값 (강제 배정 풀)

### 6.1 개요

전체 후보 상담사 간 배정 격차가 임계값 이상이면 **불균형 상태**로 판단하고,
가장 적게 배정된 상담사들만 **강제 배정 풀**로 구성합니다.

### 6.2 작동 방식

#### 조건 확인
```javascript
배정 격차 = max(활성케이스수) - min(활성케이스수)
불균형 판정 = 배정 격차 >= 임계값(기본값: 5)
```

#### 강제 풀 구성
- 불균형 상태일 때: 활성 케이스가 **최소인 상담사들만** 후보로 선정
- 정상 상태일 때: 일반 배정 규칙 적용

#### 강제 풀 내 배정
- 강제 풀 내에서도 **점수가 높은 상담사 우선**
- 점수 동점 시 **배정인원이 적은 상담사 우선**
- 최종 랜덤 선택

### 6.3 예시

**상황:**
```
상담사A: 활성케이스 12개
상담사B: 활성케이스 10개
상담사C: 활성케이스 5개
상담사D: 활성케이스 4개

배정 격차 = 12 - 4 = 8 (임계값 5 초과)
```

**결과:**
- 강제 풀 구성: **상담사D만** (최소값 4)
- 상담사D에게 우선 배정

## 7. 배정 점수 확인 표 사용법

배정 완료 후 화면에 표시되는 **배정 점수 확인 표**로 배정 과정을 상세히 확인할 수 있습니다.

### 7.1 표 구조

| 컬럼명 | 설명 |
|--------|------|
| **참여자번호** | 배정 순서 |
| **선택유형** | `fixed`(고정배정) / `random`(랜덤배정) / `forced`(강제배정) |
| **선택상담사** | 최종 배정된 상담사 ID |
| **전체 후보(점수)** | 모든 후보와 점수 <br> 예: `C001(0.850), C002(0.820), ...` |
| **상위 4명** | 점수 상위 4명 + 배정인원 + 한도만족 <br> 예: `C001(0.850, T:25, ✓), C002(0.820, T:28, ✓)` |
| **최종 후보** | 배정인원 최소인 최종 후보들 <br> 예: `C001(T:25), C004(T:25)` |
| **상담사** | 현재 행의 상담사 ID |
| **총점** | 최종 점수 (0~1) |
| **배정인원** | 전체 배정 인원 (total) |
| **한도만족** | ✓(만족) / ✗(초과) |
| **C_load** | 업무량 비용 |
| **C_fair** | 공정성 비용 (상세 툴팁) |
| **C_streak** | 연속 배정 비용 |
| **C_pace** | 진행 속도 비용 |
| **인원가중치** | 백엔드에서 받은 가중치 (0.5~2.0) |
| **근속그룹** | G1/G2/G3 |
| **선택이유** | 배정 사유 |

### 7.2 기호 설명

| 기호 | 의미 |
|------|------|
| `T:25` | Total - 전체 배정 인원 25명 |
| `✓` | 한도 만족 |
| `✗` | 한도 초과 |

### 7.3 C_fair 상세 정보

C_fair 컬럼을 클릭하면 상세 비율이 표시됩니다:
```
총점:0.2500 | 유형:0.4000, 성별:0.3000, 연령:0.2000, 특정계층:0.1500, 학력(대졸/고졸):0.3500, 경력:0.2000
```

### 7.4 배정 로직 확인 방법

**예시 1: 정상 배정**
```
상위 4명: C001(0.850, T:25, ✓), C002(0.820, T:28, ✓), C003(0.800, T:26, ✓), C004(0.790, T:25, ✓)
최종 후보: C001(T:25), C004(T:25)
선택상담사: C001
```

**분석:**
1. 점수 상위 4명 선정: C001~C004
2. 모두 한도 만족 (✓)
3. 배정인원 최소(25명): C001, C004
4. 랜덤 선택: C001

---

**예시 2: 한도 초과 상황**
```
상위 4명: C001(0.850, T:25, ✗), C002(0.820, T:28, ✓), C003(0.800, T:26, ✓), C004(0.790, T:30, ✗)
최종 후보: C002(T:28), C003(T:26)
선택상담사: C003
```

**분석:**
1. 점수 상위 4명 선정: C001~C004
2. 한도 만족 필터링: C002, C003만 통과
3. 배정인원 최소(26명): C003
4. 선택: C003

## 8. 학력 및 특성 분류 기준

### 8.1 학력 분류

시스템은 학력을 **2개 그룹**으로 분류합니다:

| 그룹 | 포함 항목 | 내부 코드 |
|------|-----------|-----------|
| **대졸 그룹** | 초대졸, 대졸, 대학원 | `college` |
| **고졸 그룹** | 고졸, 검정고시 | `high` |

**C_fair 계산 시 적용:**
- 참여자가 대졸 그룹 → 상담사의 대졸 그룹 배정 비율 확인
- 참여자가 고졸 그룹 → 상담사의 고졸 그룹 배정 비율 확인

### 8.2 연령 분류

| 구분 | 기준 | 내부 코드 |
|------|------|-----------|
| **청년** | 15세 ~ 34세 | `youth` |
| **중장년** | 35세 이상 | `middleAged` |

생년월일로부터 자동 계산됩니다.

### 8.3 참여유형

| 유형 | 설명 | 내부 코드 |
|------|------|-----------|
| **Type1** | 청년 참여자 | `type1` |

### 8.4 특정계층

## 9. FAQ

### Q1. 점수가 가장 높은 상담사가 선택되지 않았어요.

**A:** 배정은 다음 우선순위로 결정됩니다:
1. 점수 상위 4명 선정
2. 한도 만족 필터링 (랜덤 배정 모드)
3. **배정인원이 가장 적은 상담사 우선**
4. 랜덤 선택

점수가 높더라도 배정인원이 많으면 선택되지 않을 수 있습니다.

---

### Q2. "점수만 반영 배정"은 언제 사용하나요?

**A:** 다음 상황에서 사용합니다:
- 긴급하게 배정이 필요한 경우
- 한도를 일시적으로 무시해야 하는 경우
- 순수 점수만으로 최적 매칭을 원하는 경우

⚠️ **주의:** 불균형이 발생할 수 있으므로 신중하게 사용하세요.

---

### Q3. 강제 배정 풀은 왜 발생하나요?

**A:** 상담사 간 배정 격차가 임계값(기본 5명) 이상일 때 자동으로 발동됩니다.
배정이 특정 상담사에게 과도하게 집중되는 것을 방지하기 위한 안전장치입니다.

---

### Q4. 가중치는 어떻게 조정하나요?

**A:** 화면의 가중치 입력란에서 실시간으로 조정 가능합니다:
- `w_load`: 업무량 비중 (기본 0.45)
- `w_fair`: 공정성 비중 (기본 0.35)
- `w_streak`: 연속 배정 방지 (기본 0.10)
- `w_pace`: 진행 속도 (기본 0.10)

**합계는 1.0이 아니어도 됩니다.** 각 항목의 상대적 중요도를 나타냅니다.

---

### Q5. 배정 결과를 되돌릴 수 있나요?

**A:** "초기화" 버튼을 클릭하면 고정 배정을 제외한 모든 배정이 초기화됩니다.
재배정 시 랜덤 요소로 인해 결과가 달라질 수 있습니다.
