# 참여자 관리 페이지 디자인 추천

## 전제
- 현재 페이지는 AdminLTE + Bootstrap 5를 사용 중이며, 리스트 중심 UI다.
- 실사용자는 대량 데이터 확인, 빠른 필터링, 상태 파악이 핵심이다.

## 우선 추천(즉시 가치 높은 3가지)
1) **상단 KPI 요약 + 액션 영역 분리**
   - 상단에 카드 3~5개(전체/진행중/마감/이번달 신규/지점별)로 현황을 즉시 파악.
   - 카드 오른쪽에 주요 액션(엑셀 다운로드, 신규 등록, 일괄 변경)을 고정.
2) **필터/검색 영역을 “고정형 서브헤더”로 재구성**
   - 검색 조건이 긴 페이지에서 사라지지 않도록 sticky bar로 유지.
   - “기본 필터(연도/유형/상태)”와 “상세 필터(진행단계/담당자/이관 여부)”를 펼침형으로 분리.
3) **테이블 가독성 강화**
   - 상태/유형/진행단계는 칩(Chip)으로 시각적 인지 강화.
   - 중요한 컬럼(참여자명, 담당, 상태)은 왼쪽 고정.
   - 행 hover 시 액션(상세보기/수정/이관)을 인라인으로 노출.

## 레이아웃 제안
- **상단 구조**
  - 좌: 페이지 타이틀 + 간단 설명(“참여자 조회 및 관리”)
  - 우: 주요 액션 버튼
- **KPI 카드 row**(카드 간 대비, 아이콘 사용)
  - 카드 높이와 간격을 통일해 리듬감 부여.
- **필터/검색 바**
  - 2단 구성: 기본 필터(즉시 선택) + 상세 필터(아코디언).
- **테이블 영역**
  - 헤더 고정, 가로 스크롤 최소화.
  - 빈 상태/검색 결과 없음에 대한 안내 메시지와 CTA 제공.

## 스타일 방향(톤 & 무드)
- **데이터 중심, 차분하고 신뢰감 있는 톤**
- 배경은 연한 그레이 + 카드 화이트로 계층 강조.
- 포인트 컬러는 기존 AdminLTE primary를 유지하되, 상태에 맞는 색 체계(진행/경고/마감)를 명확히.

## 타이포그래피
- 제목: 20–24px, 굵기 600
- 테이블 본문: 13–14px
- 보조 정보(생년월일/내부 ID): 12px + muted color

## 컴포넌트 디테일
- **상태 칩**
  - 진행중(blue), 마감(red), 대기(gray)
- **진행단계 배지**
  - “초기/중기/후기” 색상 스펙 정의
- **행 액션**
  - hover 시 아이콘 버튼(보기/수정/이관), 평상시는 숨김
- **일괄 처리**
  - 체크박스 선택 시 상단에 “선택됨 N명” 바 고정 표시

## 상호작용/UX
- 필터 적용 시 상단에 선택된 조건을 “필터 태그”로 표시하고 한 번에 해제 가능.
- 결과 로딩 시 스켈레톤 로딩 적용.
- 행 클릭 시 우측 슬라이드 패널로 상세 정보(빠른 확인용) 제공.

## 접근성/반응형
- 테이블이 좁아지는 모바일에서는 카드형 리스트로 전환.
- 색상만으로 상태를 구분하지 않도록 텍스트 병기.

## 적용 우선순위 제안
1) KPI 카드 + 액션 정렬
2) 필터 바 고정/정리
3) 테이블 칩/고정열/hover 액션
4) 상세 패널 및 스켈레톤 로딩

## 스타일 작성 기준
- 인라인 `style` 태그 대신 전용 `.css` 파일을 생성해 적용.
- 페이지별 스타일은 `/css/{page}_*.css` 또는 `/css/{page}.css`로 관리.
