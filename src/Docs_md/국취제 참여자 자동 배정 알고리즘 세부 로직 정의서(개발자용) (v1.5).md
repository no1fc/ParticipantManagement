# 국취제 참여자 자동 배정 알고리즘 세부 로직 정의서(개발자용) (v1.5)

## 1. 목적
`praCSVtoTableJS_0.1.3.js`와 `participantRandomAssignmentMain.jsp` 기준으로 자동 배정 알고리즘의 입력/검증/산정식/저장 흐름을 개발자 관점에서 정리한다.

## 2. 관련 파일/엔드포인트
- 화면: `src/main/webapp/WEB-INF/views/participantRandomAssignmentMain.jsp`
- 프론트 로직: `src/main/webapp/js/praCSVtoTableJS_0.1.3.js`
- 저장 API: `POST /api/pra.history`

## 3. 화면 및 데이터 흐름
1) CSV 업로드 또는 직접 입력으로 참여자 목록 생성
2) 산정식 설정 카드(가중치/제한) 확인 또는 수정
3) 랜덤 배정 실행
4) 데이터 검증(옵션)
5) 저장(배정 + 히스토리 동시 저장)

## 4. CSV 처리
### 4.1 헤더 검증
- 첫 줄을 헤더로 간주
- DOM의 `.csv-column` 텍스트와 정확히 일치해야 함(순서 포함)
- 불일치 시 중단

### 4.2 행 구조
테이블 열 순서:
1) 번호(자동) 2) 상담사ID(배정 결과) 3) 참여자 성명 4) 참여유형 5) 성별 6) 생년월일 7) 모집경로 8) 경력유무(숫자) 9) 학력 10) 특정계층여부 11) 진행단계 12) 출장상담사

### 4.3 고정 배정
- 출장상담사 컬럼 값이 있으면 해당 상담사로 고정 배정
- 고정 배정 행은 `data-fixed="true"`로 표시

## 5. 검증 규칙
- 참여자 성명: 필수
- 참여유형: `1`, `2`, `1유형`, `2유형`
- 성별: `남`, `여`
- 생년월일: `YYYY-MM-DD`
- 모집경로: 필수
- 경력유무: 숫자
- 학력: 필수
- 특정계층여부: `O`, `X`, `Y`, `N`, `해당`, `해당 없음`
- 진행단계: 필수
- 출장상담사: 영문/숫자/`_`/`-`만 허용
- 출장상담사 입력 시 상담사 목록에 존재해야 유효

## 6. 자동 배정 로직
### 6.1 참여자 정규화
- 참여유형: `type1`/`type2`
- 성별: `man`/`woman`
- 특정계층: `specialGroup`/`normal`
- 연령대:
  - 입력값에 `청년`/`중장년` 포함 시 해당 값 사용
  - `YYYY-MM-DD`이면 나이 계산(34세 이하 청년)

### 6.2 상담사 상태 필드(주요)
- 기본: `name`, `employmentDate`, `total`, `max`, `current`
- 유형/성별/연령/계층: `type1`, `type2`, `man`, `woman`, `youth`, `middleAged`, `specialGroup`
- 기간 배정 카운트: `assignmentAllocationDay`, `assignmentAllocationWeek`, `assignmentAllocationTwoWeek`, `assignmentAllocationMonth`
- 연속 배정 카운트: `streak_count` 또는 `streakCount`

### 6.3 후보군 생성
제외 조건:
- `total >= max`
- `isAvailable=false` 또는 `available=false` 또는 `onDuty=false`
- 일일/7일/14일/30일 제한 초과

### 6.4 근속 그룹
- G1: 입사 3개월 이하
- G2: 입사 4~11개월
- G3: 입사 12개월 이상
- 입력 제한값: `limit-g1/g2/g3` = `week/biweek/month` 형식

### 6.5 제한 리셋
- 모든 후보가 7/14일 제한에 도달하면 1/7/14/30일 카운터를 0으로 초기화 후 재시도

### 6.6 Forced Pool
- 근속 그룹 내 `active` 편차가 `gapThreshold` 이상이면 강제 풀 생성
- 해당 그룹에서 최소 `active` 값을 가진 상담사만 후보로 선정
- 7일 배정, 연속배정 기준으로 정렬 후 선택

### 6.7 비용 함수 기반 선택
총 비용:
```
Total = w_load*cLoad + w_fair*cFair + w_streak*cStreak + w_pace*cPace
```
- cLoad: `active`와 그룹 평균 차이 제곱
- cFair: 유형/성별/연령/특정계층 분포 균형 비용
- cStreak: 연속배정(streak) 제곱
- cPace: 연간 배정(YTD)과 그룹 평균 차이 제곱

동률 처리:
1) 총 비용 오름차순
2) `active` 최소
3) 7일 배정 최소
4) 랜덤

### 6.8 결과 반영
- 배정 결과로 상담사 카운트 및 분포 업데이트
- 고정 배정도 동일하게 카운트 반영

## 7. 산정식 설정 값
- `weight-load`, `weight-fair`, `weight-streak`, `weight-pace`
- `gap-threshold`, `daily-limit`
- `limit-g1`, `limit-g2`, `limit-g3`

## 8. 저장 구조
API: `POST /api/pra.history`
```
{
  "assignments": [ { counselorID, participant, birthDate, gender, recruitmentPath, participationType, progressStage, specificClass } ],
  "dailyReports": [ { counselorID, assignmentType1, assignmentType2 } ],
  "scoringConfig": { weightLoad, weightFair, weightStreak, weightPace, gapThreshold, dailyLimit, limitG1, limitG2, limitG3 },
  "csvHistories": [ { rowNumber, counselorID, participant, participationType, gender, birthDate, recruitmentPath, hasCareer, education, specificClass, progressStage, travelCounselor } ]
}
```

서버 저장:
- `assignments`: `J_참여자관리` insert
- `dailyReports`: `J_참여자관리_일일업무보고` 오늘자 상담사별 upsert(유형1/유형2)
- `scoringConfig`: `J_참여자관리_배정산정식_히스토리` insert
- `csvHistories`: `J_참여자관리_배정CSV_히스토리` insert

## 9. 오류/예외
- CSV 헤더 불일치 시 중단
- 유효성 검증 실패 시 저장/배정 중단
- 상담사 데이터 미존재 시 배정 중단
- 미배정 존재 시 사용자 확인 후 진행
