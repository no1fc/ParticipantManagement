# 참여자 관리 시스템 - 관리자 페이지 기획 정의서 (Admin_Page_Create)

## 1. 개요
본 문서는 JobMoa 프로젝트의 통합 관리 및 상담사 배정 효율화를 위한 관리자 전용 페이지의 구성 요소와 기능을 정의합니다. 상담사가 사용하는 개별 관리 기능을 넘어, 전체 지점의 통계 확인 및 배정 알고리즘 정책을 제어하는 것에 목적을 둡니다.

## 2. 주요 기술 스택
- **Backend:** Spring Boot 3.4.1 (Java 17), MyBatis, MS SQL Server
- **Frontend:** AdminLTE v4 (Bootstrap 5.3), Vanilla JS (ES6+)
- **Real-time:** WebSocket (STOMP) - 실적 및 알림 실시간 업데이트용

---

## 3. 관리자 페이지 핵심 구성 (Menu Structure)

### 3.1. 통합 관리 대시보드 (Total Dashboard)
- **전체 KPI 요약:** 전 지점 총 참여자 수, 당월 신규 배정 건수, 전체 취업률, 목표 달성률.
- **실시간 실적 그래프:** 지점별 취업 성과 및 알선 현황 (ApexCharts 활용).
- **상태별 참여자 분포:** 진행 단계별(1~3단계, IAP 전/후) 전체 현황 파악.
- **알림 센터:** 이력서 요청 건, 미배정 참여자 발생 등 주요 이슈 실시간 노출.

### 3.2. 참여자 전체 조회 및 이관 관리 (Participant Management)
- **고급 필터링:** 지점별, 전담 상담사별, 참여 유형별 필터링 기능.
- **일괄 배정 및 이관:** 상담사 퇴사 또는 지점 이동 시 참여자 대량 이관 기능 (`transferPage.jsp` 확장).
- **데이터 보정:** 잘못 입력된 참여자 정보(생년월일, 유형 등)의 마스터 권한 수정.

### 3.3. 상담사 및 지점 관리 (User & Branch Management)
- **지점 설정:** 지점별 목표 인원, 가용 상담사 수, 지역 설정.
- **상담사 권한 제어:** 상담사 계정 생성, 비활성화, 비밀번호 강제 초기화.
- **상담사 가용성 상태:** 배정 알고리즘에 영향을 주는 상담사별 ON/OFF 상태 및 근속 그룹(G1~G3) 관리.
