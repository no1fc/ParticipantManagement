# 참여자 자동배정 알고리즘 사용자 워크플로우

## 1. 사용자 및 역할
- 관리자(고용센터): 배정 실행, 정책 관리, 결과 검수
- 상담사: 배정 대상 확인 및 상담 진행
- 시스템: 자동 배정 및 상태 갱신

## 2. 기본 워크플로우 (배치 배정)
1. 관리자가 참여자 리스트를 준비한다.
2. 시스템이 참여자 난이도를 계산하고 난이도 높은 순으로 정렬한다.
3. 시스템이 참여자 1명을 선택하고 후보군을 생성한다.
4. 후보군이 없으면 해당 참여자를 대기열로 이동한다.
5. 후보군이 있으면 불균형 트리거를 확인한다.
6. 불균형일 경우 강제 배정 후보군에서 상담사를 선택한다.
7. 균형일 경우 비용 함수로 상담사를 선택한다.
8. 동점이면 `active_case_count` 적은 순 → `assigned_7d` 적은 순 → 랜덤 순으로 결정한다.
9. 배정 결과를 확정하고 상담사 상태를 즉시 갱신한다.
10. 다음 참여자로 이동해 3~9단계를 반복한다.

## 3. 단건 배정 워크플로우
1. 관리자가 단일 참여자 배정을 요청한다.
2. 시스템이 후보군을 생성한다.
3. 불균형 여부 확인 후 강제 배정 또는 비용 함수 배정을 수행한다.
4. 결과를 확정하고 상태를 갱신한다.

## 4. 예외 및 경계 케이스
- 후보군이 비어 있음: 참여자를 대기열로 이동하고 재시도 정책에 따라 재배정한다.
- 상담사 상태 변경(근무 OFF): 배정 직전 상태를 재확인하고 후보군을 재구성한다.
- 대량 배정 중 상태 급변: 각 배정마다 상태를 갱신해 다음 배정에 반영한다.

## 5. 관리자 확인 포인트
- 배정 전: 근무 가능 상태와 기간별 한도 설정 확인
- 배정 중: 불균형 트리거 발생 여부와 후보군 로그 확인
- 배정 후: 경력 그룹 편차와 속성 편중 지표 확인

## 6. 상담사 워크플로우
1. 상담사는 배정 결과를 확인한다.
2. 참여자 속성과 진행단계를 확인한다.
3. 상담을 진행하고 상태를 업데이트한다.
4. 상담 진행 상태가 변경되면 `active_case_count`에 반영된다.

## 7. 산출물 및 기록
- 배정 결과: 참여자-상담사 매핑 목록
- 로그: 후보군, 비용 점수, 동점자 처리 근거, 상태 갱신 내역
- 지표: 경력 그룹 편차, 속성 분산, 연간 페이스 편차
