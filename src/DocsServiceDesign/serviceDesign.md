## 요약
본 서비스 설계서는 국민취업지원제도 참여자를 위한 맞춤형 채용공고 추천 시스템 구축을 목적으로 작성되었습니다. 고용24의 `Open API(개방형 인터페이스, 오픈 에이피아이)`를 하루 4회 `Scheduler(자동 실행 프로그램, 스케줄러)` 방식으로 자체 데이터베이스에 동기화하여 공공 서버의 트래픽 부담을 최소화하고, 안정적인 서비스를 제공한다는 점을 강조합니다.

### 1. 서비스 개요 및 타겟층
* **서비스 목적:** 고용24의 검증된 채용 데이터를 활용하여 국민취업지원제도 참여자들의 구직 활동을 효율적으로 지원합니다.
* **핵심 타겟층:** 국민취업지원제도 1유형 및 2유형 참여자.
* **제대 가치:** 무분별한 정보 탐색 시간을 줄이고, 참여자의 역량과 취업 활동 계획에 부합하는 일자리를 적시에 제공합니다.

### 2. 단계별 구축 목표 및 주요 기능
장기적인 비전을 제시하되, 현재 구현 가능한 범위를 명확히 나누어 설명하는 것이 신뢰도를 높입니다.
* **1단계 (현재 목표 - 채용공고 통합 리스트 구현):** 고용24 API 연동을 통한 기본 채용공고 리스트 표출 페이지를 개발합니다. 지역, 직무, 고용 형태 등 필수 조건 검색 기능을 우선적으로 지원합니다.
* **2단계 (추후 목표 - 맞춤형 추천 알고리즘 도입):** 국민취업지원제도 참여자의 이력서, 보유 자격증, 희망 직무 데이터를 기반으로 맞춤형 기업을 제안하는 큐레이션 서비스를 고도화합니다.

### 3. 트래픽 관리 및 데이터 동기화 정책 (핵심 심사 기준)
API 심사 시 가장 중요하게 보는 '서버 부하 방지 대책'을 명확히 서술합니다.
* **호출 방식:** 사용자가 화면을 조회할 때마다 실시간으로 고용24 API를 호출(Direct Call)하지 않습니다.
* **동기화 주기:** `Spring Boot(웹 프레임워크, 스프링 부트)`의 스케줄링 기능을 활용하여 매일 총 4회(00:00, 06:00, 12:00, 18:00) API를 일괄 호출합니다.
* **데이터 적재:** 호출된 최신 채용 데이터는 자체 `Database(데이터베이스, 데이터베이스)`에 저장(Insert/Update)되며, 사용자에게는 자체 서버에 적재된 데이터를 가공하여 보여줍니다. 이를 통해 공공 데이터 서버의 트래픽 한도를 엄격히 준수합니다.

### 4. 시스템 구조도 (System Architecture)
실제 개발 환경을 명시하여 실현 가능성을 증명합니다.
* **Client(클라이언트, 클라이언트) 영역:** `JSP(자바 서버 페이지, 제이에스피)`를 활용하여 사용자 친화적인 웹 화면을 구성합니다.
* **Server(서버, 서버) 영역:** `Spring Boot(웹 프레임워크, 스프링 부트)` 기반의 `REST API(웹 서비스 구조, 레스트 에이피아이)` 서버를 구축하여 데이터 통신을 담당합니다.
* **데이터 처리 체계:** `MyBatis(데이터베이스 연동 기술, 마이바티스)`를 사용하여 비즈니스 로직과 `MSSQL(관계형 데이터베이스, 엠에스에스큐엘)` 간의 데이터를 안정적이고 빠르게 처리합니다.

### 5. 데이터 표출 및 관리 규정
* **출처 명시:** 공공데이터 활용 지침에 따라, 모든 채용공고 상세 페이지 상/하단에 **"본 정보는 고용24에서 제공된 정보입니다"**라는 문구와 함께 원본 링크를 명시합니다.
* **데이터 최신성 및 파기:** API 호출 주기(6시간)에 맞춰 마감된 공고는 화면에서 즉각 블라인드 처리하며, 불필요한 지난 채용 데이터는 주기적으로 폐기합니다.

## 상세 분석: 채용공고 검색 화면 설계(UI) 요건

### 1. 화면 구성 개요
* **페이지명:** 채용공고 검색 (JobMoa)
* **화면 구조:** 1. 상단: 헤더 및 서비스 타이틀 (Hero Section)
    2. 좌측/중앙: 8개 그룹으로 분류된 상세 검색 필터 패널
    3. 하단: 비동기 방식(AJAX)으로 렌더링되는 카드형 검색 결과 및 페이지네이션
    4. 팝업: 지역 및 직종 선택용 2단 모달(Modal) 창

### 2. 상세 검색 필터 요건 (Search Panel)
고용24 데이터와 1:1 매칭되는 세밀한 필터링을 제공합니다.

* **[필터 1] 지역 및 직종 (다중 선택 및 모달 UI)**
    * **근무 지역:** 대분류(시/도) $\rightarrow$ 소분류(시/군/구) 2단 선택. 최대 5개 선택 제한.
    * **희망 직종:** 대분류 $\rightarrow$ 상세 직종 2단 선택. 최대 10개 선택 제한.
    * *UX 특징:* 모달 내 검색창을 제공하여 지역/직종명을 빠르게 텍스트로 검색할 수 있도록 지원.

* **[필터 2] 고용 형태 및 급여 (조건부 활성화 UI)**
    * **고용/임금 형태:** 정규직, 계약직 등 형태 선택. (Dropdown)
    * **급여 범위:** 임금 형태(연봉, 월급, 시급 등)를 선택했을 때만 최소/최대 급여 입력창(Number Input)이 활성화되도록 제어하여 입력 오류 방지.

* **[필터 3] 학력 및 경력 (상호 배타적 제어)**
    * **학력:** 체크박스 다중 선택. 단, '학력무관' 선택 시 다른 항목은 자동 선택 해제.
    * **경력:** 신입/경력/관계없음. '경력' 항목 체크 시에만 **경력 기간(최소~최대 개월 수)**을 입력하는 UI가 슬라이드 다운(Slide Down) 형태로 노출.

* **[필터 4] 기업 형태 및 상세 조건**
    * **기업 형태:** 대기업, 벤처기업, 공공기관 등 다중 체크박스.
    * **상세 조건:** 사원 수, 우대조건(일반/컴퓨터), 근무 형태(주 5일 등) 드롭다운 필터.

* **[필터 5] 부가 조건 및 키워드**
    * **정렬/건수:** 등록일 기준(오늘, 1주 이내 등), 정렬 방식(최신순/오래된순), 페이지당 노출 건수(10~100건) 설정.
    * **키워드:** 직종명, 회사명 등을 자유롭게 검색하는 텍스트 폼 (엔터키 이벤트 바인딩).

### 3. 검색 결과 표출 요건 (Results Section)
* **비동기 렌더링(AJAX):** 검색 버튼 클릭 시 화면 새로고침 없이 자체 API(`/recruitmentInformation/search`)를 호출하여 결과를 카드 형태로 동적 렌더링.
* **로딩 및 예외 처리:** API 통신 중에는 Spinner(로딩 링)를 노출하고, 결과가 없으면 'Empty State(빈 화면 안내)' UI를 표출. 결과가 있으면 초기 진입 시에는 최신 취업정보 UI를 표출
* **리스트형 UI (Job Card):**
    * 회사명, 공고명, 태그(지역, 직종, 고용형태), 급여(아이콘 강조), 학력/경력 뱃지(Badge), 마감일 등 핵심 정보를 시각적으로 그룹화.
    * **데이터 출처 명시 (심사 필수 항목):** 고용24 상세 페이지 원본 링크를 `상세보기` 버튼(`target="_blank"`)에 연결. *(상세정보 URL로 이동)*

## 요약
제공해주신 API 출력 결과 표를 바탕으로 채용공고 데이터를 저장할 `Table Schema(테이블 구조, 테이블 스키마)`를 재구성했습니다. API 응답의 메타데이터(총 건수 등)를 제외하고, 실제 개별 채용공고 항목(`wanted` 객체 내부 데이터)을 중심으로 데이터 무결성과 검색 효율성을 고려하여 컬럼명과 데이터 타입을 정의했습니다.

한글 데이터가 많이 포함되는 공공데이터 특성과 안정적인 시스템 운용을 위해 가변 길이 문자열에는 `NVARCHAR`를, 고유 식별자나 영문/숫자 코드에는 `VARCHAR`를 적용하여 데이터 공간을 최적화했습니다.

### 1. API 응답 메타데이터 (DB 적재 제외 항목)
* `wantedRoot`, `total`, `startPage`, `display`, `wanted` 항목은 `API(응용 프로그램 인터페이스, 에이피아이)` 통신 및 `Pagination(페이지 나누기, 페이지네이션)` 처리를 위한 메타데이터입니다.
* 이 항목들은 `Spring Boot(웹 프레임워크, 스프링 부트)` 로직 단에서 파싱 용도로만 사용하며, 실제 채용공고 테이블에는 저장하지 않습니다.
