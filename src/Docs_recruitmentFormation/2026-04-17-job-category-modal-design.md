# 희망직무 모달 선택기 설계 스펙

**작성일**: 2026-04-17
**상태**: 구현 완료 (2026-04-17)
**목적**: 희망직무 등록 시 select 드롭다운을 3컬럼 모달 검색/선택 UI로 교체

---

## 1. 요구사항

- "희망직무 추가" 버튼 클릭 시 모달 열림
- 모달 안에서 대분류(13개) > 중분류 > 소분류(직무) 3단계를 3컬럼 패널로 탐색
- 상단에 검색 input: 전체 OCC_DATA에서 직무명 키워드 검색
- 소분류 1건 선택 후 확인 → 위시리스트에 1건 추가
- 추가된 항목의 직무명(jobWant)은 사용자가 텍스트로 수정 가능
- 기존 SortableJS 드래그앤드롭 순서 변경 유지
- 최대 5건, 최소 1건

## 2. UI 구조

### 2.1 모달 레이아웃

```
┌─────────────────────────────────────────────────┐
│  희망 직무 선택                            [닫기] │
├─────────────────────────────────────────────────┤
│  🔍 직종명 검색...                               │
├──────────┬──────────────┬───────────────────────┤
│ 대분류    │ 중분류        │ 직무(소분류)           │
│          │              │                       │
│ ● 경영·  │ ● 행정·경영· │ ○ 의회의원·고위공무원  │
│   사무·  │   금융·보험   │ ○ 기업 고위임원        │
│   금융·  │   관리직      │ ○ 정부행정 관리자      │
│   보험   │              │ ○ 경영지원 관리자      │
│          │ ○ 경영지원    │                       │
│ ○ 연구   │   사무       │                       │
│   및 공학│              │                       │
│   기술   │ ○ 회계·경리  │                       │
│          │   사무       │                       │
│ ○ 교육·  │              │                       │
│   법률·  │              │                       │
├──────────┴──────────────┴───────────────────────┤
│                          [취소]  [선택 확인]      │
└─────────────────────────────────────────────────┘
```

- **좌측 패널**: 대분류 13개 리스트. 클릭 시 active 표시, 중앙 패널 갱신.
- **중앙 패널**: 선택된 대분류의 중분류 리스트. 클릭 시 active 표시, 우측 패널 갱신.
- **우측 패널**: 선택된 중분류의 소분류(직무) 리스트. 라디오 버튼 방식 (1건만 선택).
- **검색**: 입력 시 3컬럼 대신 단일 검색결과 리스트 표시. `[대분류] 중분류 > 직무명` 형태. 결과 클릭 시 해당 직무 선택.
- **확인 버튼**: 선택된 소분류가 있을 때만 활성화.

### 2.2 위시리스트 아이템 (모달 확인 후)

```
┌─────────────────────────────────────────────────────────┐
│ ≡ [1] 경영·사무·금융·보험 > 행정·경영·금융·보험 관리직     │
│      직무명: [경영기획 부서장_________________] [변경][삭제]│
└─────────────────────────────────────────────────────────┘
```

- `≡`: 드래그 핸들 (SortableJS)
- `[1]`: 순위 번호 (자동 재계산)
- 대분류 > 중분류: 읽기 전용 텍스트 (변경 시 [변경] 버튼으로 모달 재오픈)
- 직무명 input: 기본값은 선택한 소분류 name, 사용자 편집 가능
- [변경]: 해당 항목의 모달을 다시 열어 소분류 변경
- [삭제]: 해당 항목 삭제 (최소 1건 시 비활성)

## 3. 데이터 소스

### OCC_DATA 전역 노출

현재 `index_0.0.1.js`에만 로컬 변수로 존재하는 OCC_DATA를 `jobCategorySelectRenderText_0.0.2.js`에 `window.OCC_DATA`로 추가 노출.

구조:
```javascript
window.OCC_DATA = [
    {
        code: "01", name: "경영·사무·금융·보험",
        subs: [
            {
                code: "011", name: "행정·경영·금융·보험 관리직",
                subs: [
                    { code: "11100", name: "의회의원·고위공무원 및 공공단체임원" },
                    ...
                ]
            }, ...
        ]
    }, ...  // 13개 대분류
];
```

## 4. 검색 동작

- 검색어 입력 시 3컬럼 패널을 숨기고 단일 검색결과 리스트 표시
- OCC_DATA 전체에서 소분류 name을 대상으로 검색 (contains, 대소문자 무시)
- 결과 형식: `[경영·사무·금융·보험] 행정·경영·금융·보험 관리직 > 경영기획 부서장`
- 결과 클릭 시 해당 소분류 선택 상태로 전환 (라디오 선택)
- 검색어 삭제 시 3컬럼 패널로 복귀
- 결과 없으면 "검색 결과가 없습니다" 표시

## 5. 폼 제출 구조 (기존 유지)

```
wishJobList[0].categoryLarge  = "경영·사무·금융·보험"
wishJobList[0].categoryMid    = "행정·경영·금융·보험 관리직"
wishJobList[0].jobWant         = "경영기획 부서장"  (사용자 수정 가능)
wishJobList[0].wishRank        = 1
```

Legacy hidden fields 동기화 유지:
- `#jobCategoryLarge` ← 1순위 categoryLarge
- `#jobCategoryMid` ← 1순위 categoryMid
- `#counselJobWant` ← 1순위 jobWant

## 7. 상태 관리

- `editingItemIdx`: 현재 모달이 수정 중인 위시 아이템 인덱스 (-1이면 신규 추가)
- `selectedOccItem`: 모달에서 선택된 소분류 `{code, name, midName, largeName}`
- 모달 열기 시: 신규면 선택 초기화, 수정이면 기존 값으로 해당 항목 하이라이트

## 8. 엣지 케이스

- 최대 5건 도달 시 "희망직무 추가" 버튼 비활성화
- 최소 1건일 때 삭제 버튼 비활성화
- 검색 결과 클릭으로 선택 후 확인 가능
- 모달 닫기(X/취소) 시 선택 상태 초기화, 위시리스트 변경 없음
- 직무명 input이 빈 문자열이면 폼 제출 시 validation 경고
