### 시나리오 E: AI 추천 저장 전체 흐름

#### E-1. 알선상세정보 있는 참여자 추천 저장
- [ ] `IAP후` + 알선상세정보 있는 참여자 선택
- [ ] "AI 추천 채용정보 저장" 버튼 클릭
- [ ] 저장 중 버튼 비활성화 확인
- [ ] 저장 완료 메시지 "추천 저장 완료 (N건)" 출력 확인
- [ ] 모달 자동 새로고침 → 추천 리스트 출력 확인
- [ ] 베스트 채용정보 카드 출력 확인 (`bestJobInfo = "1"` 레코드 기준 — SQL: `= 1`)
- [ ] 추천 점수(`recommendationScore`) 및 추천 사유(`recommendationReason`) 출력 확인
- [ ] DB에 저장 결과 조회 확인

#### E-2. 알선상세정보 없는 참여자 추천 저장
- [ ] `IAP후` + `Referral.infoAlsonDetail` null 또는 빈값 참여자 선택
- [ ] "AI 추천 채용정보 저장" 버튼 클릭
- [ ] 저장 완료 확인
- [ ] 추천 리스트 출력 확인 (`bestJobInfo = "N"`, `recommendationScore = null`, `recommendationReason = null` 정상)

#### E-3. 동일 참여자 재저장 (UPSERT)
- [ ] 이미 추천 결과가 저장된 참여자를 다시 저장
- [ ] 기존 레코드 삭제(`deleteRecommendByGujikNo`) 후 재삽입 확인
- [ ] DB에서 중복 레코드 없음 확인
